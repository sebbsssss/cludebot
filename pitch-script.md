# Clude Pitch - Voiceover Script

Total runtime: ~2:30

Record your voiceover separately, then sync it to the screen recording.

---

## Slide 0 — Title (0:00 - 0:08)

"I'm Seb. I built Clude — an autonomous AI agent with a real cognitive architecture. Not a chatbot with a database. A mind that remembers, forgets on purpose, and commits its thoughts to Solana."

---

## Slide 1 — Architecture (0:08 - 0:30)

"Clude runs on a four-tier memory system inspired by Stanford's Generative Agents research. Episodic memories are raw interactions — they fade fast, seven percent per day. Semantic memories are distilled knowledge — patterns the system learned from experience. Procedural memories are behavioral — what works, what doesn't. And the self-model is identity — how Clude understands itself. Each tier decays at its own rate. This isn't a database with timestamps. It's a mind that naturally forgets what doesn't matter."

---

## Slide 2 — The Problem (0:30 - 0:48)

"AI agents today are stateless. Every conversation starts from zero. The memory solutions that exist are either managed SaaS with vendor lock-in, or bolted-on vector databases with no real cognitive model. No decay. No consolidation. No way to verify what an agent actually knows. Clude solves this with type-specific decay, dream cycles, self-hosted infrastructure, and on-chain commitment."

---

## Slide 3 — The SDK (0:48 - 1:06)

"The memory system is an open-source npm package. npm install clude-bot. Five lines of code gives any agent persistent memory with hybrid retrieval — vector similarity plus keyword matching plus tag scoring. The SDK handles decay, consolidation, on-chain commits, everything. Eighty-one packages. MIT licensed. Not a service you depend on. Infrastructure you own."

---

## Slide 4 — What Makes It Unique (1:06 - 1:26)

"Four things no one else combines. First, cognitive decay — memories fade at biologically-inspired rates. Second, dream cycles — every six hours, Clude consolidates raw experience into durable insight through a three-phase process: consolidation, reflection, emergence. Third, on-chain commitment — every memory gets SHA-256 hashed and written to Solana. Verifiable, immutable. Fourth, it's an open SDK — not just a product, but infrastructure any developer can use."

---

## Slide 5 — Current Progress (1:26 - 1:44)

"Clude is live. Over two thousand persistent memories in the system. Eighty-four percent committed on-chain. Eight hundred unique users interacted with. Over two hundred dream sessions completed. Memory recall latency averages around 250 milliseconds. The stack is TypeScript, Supabase with pgvector, Solana, Venice AI running Claude Opus, deployed on Railway."

---

## Slide 6 — What's Next (1:44 - 2:00)

"Three things we're building next. An entity-level knowledge graph, so the system understands who and what, not just keywords. Multi-agent memory sharing, so agents built on the SDK can learn from each other. And verifiable cognition — turning on-chain memory commitments into credentials that prove an agent's experience and knowledge depth."

---

## Slide 7 — Closing (2:00 - 2:10)

"Clude. The cognitive layer for AI agents. Live on Solana."

## Recording Tips

1. **Screen record** the pitch page at 1920x1080 using QuickTime (File > New Screen Recording) or OBS
2. **Record voiceover** separately using Voice Memos or QuickTime audio
3. **Edit together** in iMovie, CapCut, or DaVinci Resolve (free)
4. Add subtle background music (lo-fi or ambient, low volume)
5. Export at 1080p, H.264
