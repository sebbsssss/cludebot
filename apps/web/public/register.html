Choose how you want to run Clude. You can always switch later.

Local mode

No API key needed. Everything runs on your machine.

Setup wizard (recommended)
# Install and run the setup wizard
npm install @clude/sdk
npx @clude/sdk setup
# → Select "Local"
# → Installs MCP server for your IDE
Or install MCP directly
# One command, done
npx @clude/sdk mcp-install --local
Or add manually to Claude Desktop config
{
  "mcpServers": {
    "clude": {
      "command": "npx",
      "args": ["clude", "mcp-serve", "--local"]
    }
  }
}
For Node.js / TypeScript
# Install
npm install @clude/sdk

# Use
import { Cortex } from '@clude/sdk';
const brain = new Cortex({ local: true });
await brain.init();
await brain.store({ content: 'Hello!', type: 'episodic' });
const memories = await brain.recall({ query: 'hello' });

Memories stored in ~/.clude/memories.json. Embeddings via GTE-Small (~30MB, downloads on first run).

Self-hosted mode

No Clude API key needed. You connect directly to your own infrastructure.

What you need
Supabase — memory storage + vector index (free tier works)
Voyage AI — embeddings (voyage-3-large)
Anthropic — dream cycles + reflection (optional)
Setup wizard (recommended)
# Install and run the setup wizard
npm install @clude/sdk
npx @clude/sdk setup
# → Select "Self-hosted"
# → Enter your Supabase URL + service key
# → Enter your Voyage AI key
# → (Optional) Anthropic key for dream cycles
# → Creates .env + installs MCP server
Or configure manually
import { Cortex } from '@clude/sdk';

const brain = new Cortex({
  supabase: {
    url: process.env.SUPABASE_URL,
    serviceKey: process.env.SUPABASE_SERVICE_KEY,
  },
  embedding: {
    provider: 'voyage',
    apiKey: process.env.VOYAGE_API_KEY,
    model: 'voyage-3-large',
    dimensions: 1024,
  },
  agentId: 'my-agent',
});
await brain.init();
