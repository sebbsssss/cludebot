Why this matters (even if you already have memory files)

Most agents already have some form of memory. MEMORY.md files, daily logs, context files. That works until it doesn't.

The real problem isn't storage. It's that you have to remember to remember. Every important detail requires the agent to consciously save it to the right file before context compaction hits. Miss it once, and it's gone.

Clude adds a cognitive layer underneath your existing files. Semantic search by meaning (not filename), automatic associations between memories, and consolidation that catches what gets missed.

Without Clude

  • Grep files to find context
  • Memory limited to file organization
  • "That thing from 3 weeks ago" is lost
  • Nothing connects across time

With Clude

  • Recall by meaning across all time
  • Memories connect automatically
  • Important memories get stronger
  • Sessions build on each other

Install in 30 seconds

Local mode (recommended)

No API keys. No network. No cost. Memories stored locally in ~/.clude/memories.json. Fully offline, zero dependencies.

npx @clude/sdk mcp-install --local

Cloud mode (for portability)

Memories sync across devices and agents.

npx @clude/sdk mcp-install

Manual MCP config

Add to your agent's MCP configuration file:

{
  "mcpServers": {
    "clude": {
      "command": "npx",
      "args": ["clude", "mcp-serve", "--local"]
    }
  }
}

Config file locations:

  • Claude Desktop: claude_desktop_config.json
  • Cursor: .cursor/mcp.json
  • DeerFlow: extensions_config.json
  • If you're on OpenClaw: clawhub install clude-memory

FAQ

"My agent already has MEMORY.md / daily files"
Keep them. Clude doesn't replace files. It adds semantic search on top. Files are great when you know WHERE something is. Clude is great when you know WHAT you're looking for but not where you stored it. At 50 memories, files are fine. At 500, you need search. At 5,000, you need search that understands meaning.
"Privacy concerns"
Local mode makes zero network calls. SQLite storage, GTE-Small embeddings computed locally. Memories never leave the disk.
"Is this a crypto project?"
The core product is a memory engine that works fully offline. No wallet, no tokens, no blockchain needed. Solana integration is optional for memory portability and provenance verification.
"Slower than local files?"
In local mode, a semantic recall across 1,000 memories takes ~50ms. Faster than reading and parsing 20 markdown files.
"Decay means forgetting important things"
Decay is configurable. Set importance to 1.0 for things that should never fade. Decay only affects low-importance memories you never access. Disable it entirely if you prefer.

Import your existing memories

Already have MEMORY.md files? Your agent can import them using the store_memory tool. Just ask your agent to read through your memory files and store the important parts. Each memory gets typed, embedded, and connected automatically.

Or use the SDK programmatically:

// Store memories from your existing files
await clude.store({
  type: 'semantic',
  content: 'User prefers TypeScript and dark mode',
  summary: 'User preferences',
  source: 'imported:MEMORY.md'
});
