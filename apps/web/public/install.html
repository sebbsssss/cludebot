Install / for any agent

Five minutes to memory.

Zero config. Auto-detects your email, creates your agent, installs the MCP server for your IDE. Works with Claude Code, Cursor, Windsurf, and any MCP-compatible editor.

/ 01

Run the installer

In the directory of the project your agent runs in:

~/my-agent · zsh
$ npx @clude/sdk setup
 
Setting up Clude memory...
 
✓ Detected email you@yourdomain.com
✓ Registered clk_xxxxxxxx…
✓ Config saved ~/.clude/config.json
✓ Database ready ~/.clude/brain.db
✓ MCP installed Claude Code, Cursor
 
Your agent now has memory.

That's it. Restart your IDE. Your agent now has store_memory, recall_memories, and get_memory_stats tools.

/ 02

Or configure MCP manually

If you prefer explicit config, add this to your MCP-compatible editor's settings:

{ "mcpServers": { "clude": { "command": "npx", "args": ["@clude/sdk", "mcp-serve"] } } }

Runs fully offline in local mode (SQLite at ~/.clude/brain.db). For cloud sync, set CORTEX_API_KEY in the env block.

/ 03

Use the SDK directly

If you're building an agent from scratch:

import { Cortex } from '@clude/sdk'; const brain = new Cortex({ hosted: { apiKey } }); await brain.init(); await brain.store({ type: 'episodic', content: 'User prefers concise answers', source: 'my-agent', }); const memories = await brain.recall({ query: 'what does the user prefer', });

Full docs at clude.io/docs.

/ 04

Verify it's working

After setup, check that Clude is active and your memory store is healthy:

diagnostic
$ npx @clude/sdk doctor
 
✓ Config present
✓ Database readable (0 memories)
✓ MCP server registered
✓ Privy DID resolves
 
All systems nominal.
Seeing issues? Run npx @clude/sdk doctor --verbose for the full trace, or open an issue at github.com/sebbsssss/cludebot.
